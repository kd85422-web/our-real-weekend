# 자동 추천 설정 가이드

매주 자동으로 "이번 주 추천"을 채워주는 기능을 켜는 방법이에요.
구조는 이렇습니다.

```
(주 1회 / 또는 수동 실행)
Vercel 함수 /api/refresh
   ├─ 한국관광공사 행사 API  → 실제 행사·축제 (날짜·장소·사진)
   └─ 유튜브 Data API        → '화제' 영상
        ↓ 저장
   Supabase  __recos__ 공간
        ↓ 불러오기
   앱 홈 화면의 추천 (부부 개인 기록과는 분리)
```

준비물은 **무료 API 키 2개**예요. (유튜브는 선택)

---

## 1) 한국관광공사 키 발급 (필수)

1. [공공데이터포털 data.go.kr](https://www.data.go.kr) 가입/로그인
2. 검색창에 **국문 관광정보 서비스** 검색 →
   **"한국관광공사_국문 관광정보 서비스_GW"** 의 **오픈 API** 항목 클릭
3. **활용신청** 버튼 클릭 → 활용목적 간단히 적고 신청 (보통 즉시 승인)
4. 마이페이지 → **데이터활용 → 인증키 발급현황** 에서
   **일반 인증키 (Decoding)** 값을 복사해 둡니다.
   > ⚠️ "Encoding" 말고 **"Decoding"** 키를 쓰세요.

## 2) 유튜브 키 발급 (선택 — '화제 영상'을 넣고 싶을 때만)

1. [Google Cloud Console](https://console.cloud.google.com) 접속 → 프로젝트 생성
2. **API 및 서비스 → 라이브러리** → **YouTube Data API v3** 검색 → **사용 설정**
3. **사용자 인증 정보 → 사용자 인증 정보 만들기 → API 키** → 생성된 키 복사
   > 건너뛰면 행사 데이터만으로 추천이 채워집니다.

## 3) Vercel 환경변수 등록

Vercel 프로젝트 → **Settings → Environment Variables** 에서 아래를 추가하세요.
(Production 체크)

| 이름 | 값 |
|---|---|
| `SUPABASE_URL` | `https://qjdftiatmjzttiutqfrg.supabase.co` |
| `SUPABASE_KEY` | (config.js 에 넣은 publishable 키와 동일) |
| `TOUR_API_KEY` | 1)에서 받은 **Decoding** 키 |
| `YOUTUBE_API_KEY` | 2)의 키 (선택) |
| `HOME_LAT` | (선택) 집 위도. 기본 37.5663 (서울시청) |
| `HOME_LNG` | (선택) 집 경도. 기본 126.9779 |

저장 후, 변경을 적용하려면 한 번 **재배포**(Deployments → 최신 배포 … → Redeploy)하세요.

## 4) 첫 실행 (수동)

브라우저에서 아래 주소를 직접 엽니다 (본인 배포 주소로):

```
https://<당신의앱>.vercel.app/api/refresh
```

성공하면 이런 JSON이 보여요:

```json
{ "ok": true, "count": 14, "festivals": 12, "youtube": 2, "best": "○○축제" }
```

이제 앱을 새로고침하면 홈 추천이 **실제 행사**로 바뀝니다.
(에러 JSON이 나오면 `error` 메시지를 보고 키/환경변수를 점검하세요.)

## 5) 자동 갱신

`vercel.json` 에 cron이 들어 있어 **매주 목요일 저녁(한국시간 금요일 새벽)** 자동 실행됩니다.
주기를 바꾸려면 `vercel.json` 의 `"schedule": "0 21 * * 4"` 를 수정하세요 (UTC 기준).

> Vercel cron은 **운영(Production) 배포**에서만 동작해요. Hobby 플랜은 cron 2개까지 무료.

---

## 동작 방식 메모

- 자동 추천은 Supabase 의 **`__recos__`** 공간에 저장돼요. 부부 공유 코드(개인 기록)와 분리돼서, 추천이 갱신돼도 찜·후기·방문기록은 그대로예요.
- 추천 장소를 **찜하거나 후기를 쓰면** 그 장소가 부부 공간으로 복사돼 개인 기록으로 남아요.
- 인스타·스레드는 공식 API 제약으로 자동 수집에서 제외했어요(원하면 직접 추가 버튼으로 수동 등록 가능).
- 한국관광공사 데이터에는 거리·실내외·비용이 없어서, 거리는 집 좌표 기준으로 **대략 계산**하고 비용은 비워둡니다(추후 보완 가능).
